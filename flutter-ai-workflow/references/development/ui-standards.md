# UI 规范约束

本项目 UI 目标是记账工具：清晰、克制、可扫描、适合高频使用。

## 视觉原则

- 不做营销式落地页。
- 不使用装饰性渐变、光斑、漂浮背景元素。
- 不使用大面积单一色相堆叠。
- 卡片圆角不超过 8。
- 页面 section 不做嵌套卡片；卡片只用于独立信息块。
- 页面文字不得互相遮挡，按钮文字不得溢出。

## Token 约束

- 颜色必须优先来自 `AppColors` 或 `Theme.of(context).colorScheme`。
- 间距必须优先来自 `AppSpacing`。
- 圆角必须优先来自 `AppRadii`。
- 文案必须优先来自 `AppStrings`。
- 页面内避免裸写重复颜色、间距、圆角和业务文案。

## 布局约束

- 页面外边距默认使用 `AppSpacing.pagePadding`。
- 信息块之间默认使用 16、20、24 的间距层级。
- 固定格式元素需要稳定尺寸或明确约束，避免数据变化导致布局跳动。
- 金额和统计类内容必须易扫描，主金额突出，辅助金额弱化。

## 组件约束

- 工具类操作优先使用图标按钮，并提供语义说明。
- 表单输入必须有校验、错误提示和键盘类型。
- 删除等破坏性操作必须有二次确认。
- 空状态必须说明当前状态，而不是只留空白。

## 记账 App 当前约束

- 首页优先展示月度概览和最近记录。
- V0.1 只展示骨架和空状态，不提前实现真实交易逻辑。
- V0.2 再加入底部导航和静态多页面。
