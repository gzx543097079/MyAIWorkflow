# 提示词模板

当用户需要可复用 AI 指令时，使用这些模板。

## 开始一个版本

```text
我现在要做 Flutter 记账 App 的 V{version}。
请先读取当前版本目标，列出本版本要完成的功能、修改文件、验收标准和风险点。
```

## 判断下一版本

```text
请根据当前 Flutter 项目状态，判断我应该进入哪个版本。
要求按照 V0.1 到 V1.0 路线推进，不要提前加入云同步、登录、会员或复杂 AI 分析。
```

## 实现一个功能

```text
请基于当前 Flutter 项目结构，实现【功能名称】。
要求遵循 feature-first 架构，不引入不必要依赖，并说明修改了哪些文件。
```

## 审查代码

```text
请检查本次 Flutter 改动是否存在状态管理问题、数据持久化问题、UI 溢出风险、金额计算错误和测试缺口。
```

## 生成测试

```text
请为当前功能生成测试。
重点覆盖正常流程、空数据、非法输入、边界金额和日期筛选。
```

## 规划提交信息

```text
请根据本次改动生成一条清晰的 Git commit message。
格式使用 conventional commits。
```
