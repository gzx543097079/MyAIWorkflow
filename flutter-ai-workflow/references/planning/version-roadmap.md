# 版本路线

使用这份路线，把 Flutter 记账 App 拆成小而可验证的版本逐步推进。

## V0.1 项目初始化版

状态：已完成。详情见 `references/planning/project-status.md`。

目标：创建一个可以在 iOS 和 Android 运行的 Flutter 项目。

任务：
- 创建 Flutter 项目。
- 验证 iOS 模拟器可运行。
- 验证 Android 模拟器可运行。
- 添加 `main.dart` 和 `app.dart`。
- 添加基础主题和简单首页。
- 如有需要，初始化 Git。

完成标准：
- `flutter run` 可以运行。
- 首页可以正常渲染。
- 项目拥有清晰的基础目录结构。

## V0.2 静态 UI 版

状态：已完成。

目标：用静态数据搭建主要 App 壳。

页面：
- 首页
- 明细
- 记一笔
- 统计
- 设置

完成标准：
- 底部导航可以切换。
- 页面展示静态记账数据。
- 暂时不需要数据库。

## V0.3 领域模型版

状态：已完成。

目标：定义类型明确的记账数据模型。

模型：
- `Transaction`
- `Category`
- `TransactionType`

完成标准：
- UI 使用类型模型，而不是松散的 Map。
- 模型已经为本地持久化做好准备。

## V0.4 本地存储版

状态：已完成。

目标：把交易记录持久化到本地。

推荐选择：
- Hive：适合简单本地存储。
- Isar 或 Drift：适合更强的查询和统计。

完成标准：
- 可以新增一笔交易。
- 可以删除一笔交易。
- App 重启后仍能加载已保存记录。

## V0.5 记一笔流程版

状态：已完成。

目标：让日常记账流程真正可用。

字段：
- 金额
- 收入或支出类型
- 分类
- 日期
- 备注

完成标准：
- 用户可以保存一条合法交易。
- 表单校验可用。
- 保存后明细列表和首页汇总会更新。

## V0.6 统计分析版

状态：已完成。

目标：展示收入和支出的统计洞察。

功能：
- 月度收入
- 月度支出
- 月度结余
- 分类支出
- 每日趋势

完成标准：
- 统计数据来自真实交易记录。
- 图表可以正常渲染。
- 汇总金额与明细列表一致。

## V0.7 分类管理版

状态：已完成。

目标：允许用户自定义收入和支出分类。

功能：
- 默认收入分类
- 默认支出分类
- 新增分类
- 编辑分类
- 删除分类

完成标准：
- 用户可以管理分类。
- 记一笔流程可以使用自定义分类。

## V0.8 状态管理重构版

状态：已完成。

目标：让项目更易维护。

推荐技术：
- Riverpod
- Repository 层
- Controller 或 Notifier 层

完成标准：
- UI 不直接访问数据库。
- 状态更新可预测。
- 业务逻辑可以脱离 Widget 测试。

## V0.9 体验优化版

状态：下一步。

目标：提升真实产品使用体验。

功能：
- 空状态
- 加载状态
- 错误状态
- 删除确认
- 日期筛选
- 月份切换
- 深色模式

完成标准：
- App 在日常使用中感觉稳定。
- 非法输入有清晰处理。
- 常见屏幕尺寸没有明显布局溢出。

## V1.0 发布候选版

目标：准备成为自己可以真实使用的 App。

完成标准：
- 核心流程可用。
- iOS 和 Android 都经过测试。
- 基础测试通过。
- 数据可以正确持久化。
- 金额计算准确。
- 没有严重控制台错误。
